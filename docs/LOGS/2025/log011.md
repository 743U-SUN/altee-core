# AI実装プロンプトガイド

## admin/media/upload 機能実装完了報告

### プロジェクト概要

記事投稿時以外でも事前に画像をアップロードし、後で記事内画像や其の他の用途で再利用できるメディア管理機能を実装。シンプルかつ最低限の機能的なアプローチを採用し、段階的に拡張可能な設計で実装完了。

### 実装完了内容

#### ✅ データベーススキーマ拡張

**MediaType enum 拡張:**
```prisma
enum MediaType {
  THUMBNAIL  // 記事サムネイル
  CONTENT    // 記事内画像
  SYSTEM     // システム専用画像（ロゴ、アイコン等）
}
```

**MediaFile model 拡張:**
```prisma
model MediaFile {
  // 既存フィールド + 新規追加フィールド
  description   String?     // 画像説明（オプション）
  altText      String?     // alt属性用テキスト
  tags         Json?       // タグ配列 ["hero", "mobile", "v2"]
}
```

#### ✅ コンテナ構成

```
article-thumbnails/     # 記事サムネイル用

```

2. **ファイル形式:** 既存対応形式のみ（ico, pdf不要）
3. **アクセス権限:** ADMIN権限のみで十分
4. **タグ入力:** カンマ区切りテキスト入力
5. **フィールド:** すべてオプション項目
6. **プレビュー:** largeサイズ（既存ImageUploader使用）
