## 3. 指摘事項

### 3.1 【軽微】ブランド選択で「null」文字列を使用

**ファイル**: `app/admin/products/components/ProductForm.tsx` 行162

```tsx
<SelectItem value="null">ブランドなし</SelectItem>
```

**懸念**: `value="null"`は文字列"null"であり、実際のnullではない。フォーム送信時にバリデーションエラーになる可能性。

**提案**: 空文字列のハンドリングを明確にするか、「選択解除」ボタンを別途設ける。

**優先度**: 低（動作確認済みなら問題なし）

---

### 3.2 【軽微】CSVインポートでトランザクション未使用

**ファイル**: `app/admin/products/actions.ts` 行360-435

```typescript
export async function importProductsFromCSVAction(rows: ProductCSVRow[]) {
  for (let i = 0; i < rows.length; i++) {
    // 各行を個別に処理（トランザクションなし）
  }
}
```

**現状**: 各行が個別にDB操作され、途中でエラーが発生しても一部が挿入される。

**提案**: 以下のいずれかを検討
1. `prisma.$transaction()`で一括処理（全成功or全失敗）
2. 現状維持（部分成功を許容し、エラー行のみ報告）← 現在の実装

**優先度**: 低（現在の実装でも実用上は問題なし。エラー報告が適切に行われている）

---

### 3.3 【情報】削除時の確認が二重にならない設計

**観察**: DeleteCategoryButtonとDeleteProductButtonでは、クライアント側でダイアログ確認 → サーバー側で削除前チェックの二段構成。

**評価**: 良い設計。UXとデータ整合性の両方を考慮している。
