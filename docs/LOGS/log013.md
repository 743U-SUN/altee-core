### 概要

admin/attributes 統合管理システムの実装計画。カテゴリとタグ機能を実装し、将来的な属性拡張に対応できる設計とする。

### 実装方針

- **YAGNI**: 必要最小限から開始、段階的拡張

- **拡張性**: 将来の属性追加に対応できる設計
- **一貫性**: 既存のadmin/articles パターンに従った実装

### データベーススキーマ

#### 新規追加モデル

```prisma
model Category {
  id          String    @id @default(cuid())
  name        String    @unique
  slug        String    @unique
  description String?
  color       String?   // UI表示用の色コード (#RRGGBB)
  order       Int       @default(0) // 表示順序
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
  
  // リレーション
  articles    ArticleCategory[]
  
  @@index([order])
  @@index([name])
  @@map("categories")
}

model Tag {
  id          String    @id @default(cuid())
  name        String    @unique
  slug        String    @unique
  description String?
  color       String?   // UI表示用の色コード (#RRGGBB)
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
  
  // リレーション
  articles    ArticleTag[]
  
  @@index([name])
  @@map("tags")
}

// 記事とカテゴリの中間テーブル（多対多）
model ArticleCategory {
  id         String   @id @default(cuid())
  articleId  String
  categoryId String
  article    Article  @relation(fields: [articleId], references: [id], onDelete: Cascade)
  category   Category @relation(fields: [categoryId], references: [id], onDelete: Cascade)
  createdAt  DateTime @default(now())
  
  @@unique([articleId, categoryId])
  @@index([articleId])
  @@index([categoryId])
  @@map("article_categories")
}

// 記事とタグの中間テーブル（多対多）
model ArticleTag {
  id        String   @id @default(cuid())
  articleId String
  tagId     String
  article   Article  @relation(fields: [articleId], references: [id], onDelete: Cascade)
  tag       Tag      @relation(fields: [tagId], references: [id], onDelete: Cascade)
  createdAt DateTime @default(now())
  
  @@unique([articleId, tagId])
  @@index([articleId])
  @@index([tagId])
  @@map("article_tags")
}
```

#### トランザクション処理
記事の作成・更新時にカテゴリ・タグのリレーションもトランザクションで同時処理：
```typescript
const article = await prisma.$transaction(async (tx) => {
  // 記事作成/更新
  const newArticle = await tx.article.create({ ... })
  
  // カテゴリ・タグの関連も同時作成
  if (categoryIds.length > 0) {
    await tx.articleCategory.createMany({ ... })
  }
  if (tagIds.length > 0) {
    await tx.articleTag.createMany({ ... })
  }
  
  return newArticle
})
```

#### インライン作成機能
記事作成時にDialogでカテゴリ・タグを即座に作成可能：
```typescript
// CategoryTagSelector.tsx でのインライン作成
const handleCreateCategory = async () => {
  const formData = new FormData()
  formData.append('name', newCategoryName.trim())
  const result = await createCategory(formData)
  
  // 作成と同時に選択状態にする
  if (result.success) {
    setCategories([...categories, result.category])
    onCategoriesChange([...selectedCategoryIds, result.category.id])
  }
}
```

#### 削除保護機能
使用中のカテゴリ・タグは削除不可：
```typescript
// 記事に紐づいている場合は削除不可
if (category.articles.length > 0) {
  throw new Error(`このカテゴリは${category.articles.length}件の記事で使用されているため削除できません`)
}
```
