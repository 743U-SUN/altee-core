# MEMO

  1. それぞれのユーザーは3つの個別ページを作成できる。
  2. セクションにNEWSセクションを追加する。
  3. NEWSページを作成する。
  4. ヘッダーとフッターに追加する。
  5. admin/articlesに記事作成のあれこれがある。これを参考に。


## NEWSページについて([handle]/news)

  ・NEWSページは、縦に3つカードを並べるだけのシンプルな構造。
  ・それぞれの個別ページは[handle]/news/slug,の形で表示。  

## DashboardNewsページについて(dashboard/news)

  ・admin/articlesを参考に、ユーザー向け記事作成ページを作る。
  ・1ユーザー3つまで作成可能。
  ・並べ替え可能にする。

## NEWS作成ページについて(dashboard/news/new)

  ・個別ページはサムネイル（OGP画像）とタイトル・本文。
  ・画像は1枚までとする。（サムネイル画像をトップに置くので合計2枚）
  ・本文はMarkdown形式で書ける。  
  ・YoutubeはOK
  ・[youtube=動画ID]で動画を埋め込めるようにする。
  ・components/user-profile/sections/YoutubeSection.tsxで埋め込んだ動画のように、遅延読み込みが行われる形にしたい。
  ・動画IDはURLでも可。そこから動画IDを抽出する。
  ・[image]で画像を埋め込めるようにする。
  ・本文編集のツールバーにある画像追加アイコンをクリックしたときにも[image]がでるように。本来の[test-alt](https://cdn.altee.me/article-images/2026/03/1772416421185_c3b11a73.webp)の形にはしない。altは不要。
  ・画像はあらかじめユーザーがアップロードした画像を使用する。
  ・画像が登録されていないのに[image]と入力された場合のことを考える。（何も出力されないようにしよう。）

## NEWSセクションについて（dashboard/profile-editor）

  ・PCの場合は横に並べて3つ表示する。（画像を上に、タイトルを下にするカード型Youtube風）
  ・モバイルの場合は縦に並べて3つ表示する。（画像を左に、右にタイトルのカード型）
  ・2つの場合は2つを均等表示、1つの場合は中央に一つだけ表示する。

## Adminで管理

  それぞれのユーザーの個別ページをAdminが強制的に非表示にできるようにする。
  Adminのユーザー管理ページ(admin/users/[userid])に記事の一覧と強制非表示ボタンを作る。

## ヘッダーとフッター

  ・dashboard/profile-editorのVisibilityにNewsページを表示するかどうかの項目を追加する。非表示に設定された場合、ヘッダーとフッターからNewsページ（[handle]/news）へのリンクを削除する。それぞれ個別のnews記事は非表示にしない。
