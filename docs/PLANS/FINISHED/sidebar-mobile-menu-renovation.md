## 🎯 目的

PCサイドバーとモバイルフッターメニューに、複数のメニューセットを切り替える機能を追加する。
一般ユーザー向けの2つのメニューと、ログインユーザー専用のメニューを矢印ボタンで切り替え可能にする。

## 🖥️ PC版サイドバーの設計

### レイアウト構成

```
┌─────────────────────┐
│   [🏠 Home]         │ ← 常に一番上に固定
├─────────────────────┤
│                     │
│ 【一般メニュー1】    │
│  - VTuber一覧       │
│  - グループ一覧      │
│  - 関連記事         │
│                     │
│ ─────────────────   │ ← 軽い仕切り線
│                     │
│ 【一般メニュー2】    │
│  - PCビルダー       │
│  - 役立つアイテム    │
│                     │
├─────────────────────┤
│   [→]              │ ← ログインユーザーのみ
└─────────────────────┘    一番下に配置
```

### 設計のポイント

1. **ホームボタン**: 常に一番上に固定（左上=ホームの定石）
2. **一般メニュー1と2**: 常時表示、セクション分けで視覚的に区別
3. **矢印ボタン**:
   - ログインユーザーのみ表示
   - 一番下に配置（「さらに見る」の示唆）
   - 押すとユーザーメニューに切り替わる
4. **ユーザーメニュー表示時**: 一般メニューが非表示になり、ユーザーメニューのみ表示

## 📱 モバイルフッターメニューの設計

### レイアウト構成

```
┌────┬────┬────┬────┬────┐
│ 🏠 │ 📺 │ 👥 │ 📰 │ → │
└────┴────┴────┴────┴────┘
  ↑    ↑(可変エリア)↑   ↑
 左端                   右端
固定                    固定
```

### 設計のポイント

1. **固定位置**:
   - 左端: ホームアイコン（常に固定）
   - 右端: 矢印アイコン（メニュー切り替え）

2. **可変エリア**:
   - 中央部分: 現在のメニューに応じたアイコン
   - 最大4個まで表示可能
   - メニューによってアイコン数は可変（3個、4個など）
   - アイコン幅は個数に応じて自動調整

3. **レスポンシブ**:
   - 画面幅に応じてアイコンサイズを調整
   - タブレット・スマホ両対応

## 🔄 メニュー切り替えの動作仕様

### 矢印ボタンの動作

- **アイコン**: 右矢印 (→) のみ
- **動作**: 押すたびに次のメニューへループ

#### ゲストユーザーの場合
```
一般メニュー1 → 一般メニュー2 → 一般メニュー1 → ...
```

#### ログインユーザーの場合
```
一般メニュー1 → 一般メニュー2 → ユーザーメニュー → 一般メニュー1 → ...
```

### 初期表示ロジック

ページURLに応じて、適切なメニューを自動選択:

| アクセスページ | 初期表示メニュー |
|--------------|----------------|
| `/vtubers`, `/groups`, `/articles` | 一般メニュー1 |
| `/pc`, `/items` | 一般メニュー2 |
| `/dashboard/*` | ユーザーメニュー |
| その他 | 一般メニュー1（デフォルト） |

**⚠️ 注意**: ルートパスは仮のものです。

### 状態保持

- **初回アクセス時**: ページURLに応じた適切なメニューを表示
- **手動切り替え後**: ユーザーが選択したメニューを保持
- **ページ遷移時**: 選択状態を維持（自動切り替えしない）

## 🎨 UI/UXの設計方針

### PC版サイドバー

1. **視線の流れ**: 上（ホーム）→ 下（矢印）の自然な流れ
2. **セクション分け**: 一般メニュー1と2の間に軽い視覚的区切り
3. **切り替え時**: スムーズなアニメーション（フェードまたはスライド）
4. **アイコン**: lucide-reactを使用

### モバイルフッター

1. **固定配置**: 画面下部に常時固定
2. **タップ領域**: 十分なサイズ（最小44x44px）
3. **視覚的フィードバック**: アクティブなページのアイコンをハイライト
4. **アニメーション**: メニュー切り替え時にスムーズな遷移

### 共通

- **一貫性**: PCとモバイルで同じメニュー構造
- **直感性**: 矢印で「他のメニューがある」ことを示唆
- **アクセシビリティ**: キーボード操作対応、ARIAラベル設定

## 🛠️ 技術的な検討事項

### 状態管理

以下のいずれかの方法を検討:

1. **React Context**: グローバルなメニュー状態管理
2. **URL State (nuqs)**: URLパラメータでメニュー状態を管理
3. **LocalStorage**: ブラウザに状態を永続化

**推奨**: React Context + LocalStorage の組み合わせ
- Context: リアルタイム状態管理
- LocalStorage: セッション間での状態保持

### 既存実装の活用

現在の実装を最大限活用:

- `components/layout/Sidebar.tsx` - 既存のサイドバーコンポーネント
- `components/layout/MobileFooter.tsx` - 既存のフッターコンポーネント
- `lib/layout-config.ts` - レイアウト設定システム
- `components/ui/sidebar.tsx` - shadcn/ui Sidebarプリミティブ

### 新規作成が必要なコンポーネント（予想）

1. **MenuSwitcher**: 矢印ボタン + メニュー切り替えロジック
2. **MenuContext**: メニュー状態管理用Context
3. **useMenu**: メニュー操作用カスタムフック
4. **メニュー設定ファイル**: 各メニューの定義（アイコン、ラベル、パスなど）
