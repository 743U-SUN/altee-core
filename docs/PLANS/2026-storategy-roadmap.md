# 2026 Strategy Roadmap: altee-core

## 1. ビジョン (Vision)
**「VTuber・配信者のための、名刺代わりの『最強の個人サイト』作成ツール」**

既存のSNSリンク集（Linktree等）を超え、個性を表現できる「ホーム」を提供する。
個人勢・企業勢問わず、活動者の「名刺」となり、さらに「グループ」としての活動も支援するプラットフォームを目指す。

## 2. コア機能 (Core Features)

### A. 個人プロフィール機能 (Priority: High)
*現状：50〜60%完成*
- **基本プロフィール**: 名前、立ち絵、SNSリンク
- **PCスペック掲載**: CPU, GPU, メモリなどの環境紹介 (活動者の定番)
- **使用アイテム/デバイス**: マイク、カメラ、オーディオIFなど (アフィリエイト連携の可能性も)
- **FAQ / Q&A**: 「よくある質問」を掲載するページ
- **カスタマイズ**:
    - ベースはシンプルで使いやすく
    - テーマ選択機能により、ユーザーが個性を出せるようにする (可愛い、クール、サイバー、etc.)

### B. グループ機能 (Priority: High - 2026 Milestone)
*現状：未着手*
- **ハイブリッドな運営形態**:
    - **「事務所・ユニット」型**: オーナーが管理、招待・承認制。
    - **「コミュニティ・サークル」型**: 誰でも参加可能、または緩やかな承認制。趣味の集まり（例：モンハン部）など。
- **マルチ所属**: ユーザーは複数のグループに参加可能。
- **グループページ (Component-based)**:
    - 個人ページと同様に「セクション」を追加して構築する形式。
    - **メンバーリスト**: 所属メンバーのキャラクター画像を一覧表示。
    - **活動内容**: テキスト、画像、動画などで活動を紹介。
    - **募集要項**: 新規メンバー募集などのセクション。

## 3. ロードマップ (Roadmap)

### Phase 1: 個人プロフィール機能の完遂 & 基礎構築 (Q1 - Q2)
*「まずは個人の表現力を最大化する」*

- [-] **[Feature] 個人プロフィール機能の拡充**:
    - [-] **FAQセクション/ページ**:
        - アコーディオン形式などで「よくある質問」を表示。
        - **ガイドライン**や**依頼受付（Commission）**の条件もここでカバーする方針。
    - [-] **週間スケジュール (Section)**:
        - 配信予定などを表示するセクション。
    - [-] **活動年表 (Timeline) セクション**:
        - 縦ラインに沿って活動の歴史を表示するセクション。
    - [-] **動画ギャラリー (YouTube Carousel) セクション**:
        - メイン動画1つを大きく表示し、下に他の動画をカルーセル表示する形式。既存の「単体動画セクション」とは別に用意。
    - [-] **円グラフ / ステータス (Circular Stat)**:
        - 配信ジャンル比率やパラメータ（STR/INTなど）を可視化。デモにある `CircularStat` を採用。
    - [-] **各種リンク (Icon / List)**:
        - シンプルなSNSアイコン列 (`SimpleLinks`) と、説明付きリスト形式 (`LinkList`) の2パターンを追加。
    - [-] **画像セクション (Constraint)**:
        - 以下の3タイプを用意し、**各タイプにつき1つまで**追加可能とする（最大3セクション）。
            - [-] **Hero (1カラム)**: 大きなバナー画像 + テキスト/リンク
            - [-] **2 Column Grid**: 画像2枚並び
            - [-] **3 Column Grid**: 画像3枚並び
    - [-] **見出し・構造 (Structural)**:
        - [-] **見出し (Header)**: デザインバリエーションを追加（シンプル、リボン、アンダーライン等）。
        - [-] **余白 (Spacer)**: セクション間の調整用。
        - [-] **区切り線 (Divider)**: 装飾ライン。
    - [-] **テーマ切り替え基盤**: ユーザーが「かわいい」「クール」などを選べるように。

### Phase 2: コミュニティ基盤 & ユーザー検索 (Q2 - Q3)
- [ ] **ユーザーお知らせページ (Custom Pages)**:
    - [ ] **ページ作成制限**: 1ユーザーにつき最大3ページまで（「イベント告知」「詳細ルール」などの固定用途を想定）。
    - [ ] **画像制限**: 容量対策のため、1ページにつき画像は1〜2枚程度に制限、または外部サービス埋め込み推奨。
    - [ ] ブログ機能（記事一覧など）は実装せず、あくまで「固定ページ」としての提供に留める。
- [ ] **ユーザー検索・一覧機能 (Discovery)**:
    - [ ] **コラボ募集ステータス**: 「募集中」「条件付き」「不可」などで絞り込み。
    - [ ] **ゲーム/ハードウェア**: 「APEXをプレイ中」「Switch持ち」などでユーザーを検索。
- [ ] **グループ機能**:
    - [ ] **データベース設計**:
        - [ ] User - Group - Membership (Role: Owner, Admin, Member)
        - [ ] Groupの設定 (Public/Private, Join condition)
- [ ] **グループ作成・編集 UI**:
    - [ ] 個人ページのエディタを流用・拡張した直感的な編集画面。
- [ ] **グループページ (Public View)**:
    - [ ] メンバー一覧セクションの実装。

### Phase 3: パフォーマンス & 最適化 (Q4)
- [ ] **画像配信最適化**:
    - [ ] アップロード時の自動圧縮 / リサイズ処理
    - [ ] 適切な容量制限の実装
- [ ] **SEO強化**: 各プロフィールページのメタデータ最適化

### Sub Track: リファクタリング & メンテナンス (Ongoing)
*「機能開発の合間や別ツール(Claude Code等)を活用して実施」*
- [ ] ディレクトリ構造の整理
- [ ] 不要ファイルの削除

### Future / Nice-to-have (バックログ)

- [ ] **サムネイル作成ツール**: 配信待機画面やサムネをブラウザ上で作成
- [ ] **高度なデザインカスタマイズ**: CSS編集など上級者向け機能
